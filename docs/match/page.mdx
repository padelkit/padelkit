# Match

The `Match` class is the top-level entity representing a padel match. It encapsulates both the metadata (teams, duration, location) and the state management logic by holding a `MatchScoreHistory` instance. {{ className: 'lead' }}

## Properties

<Properties>
  <Property name="history" type="MatchScoreHistory">
    The underlying [`MatchScoreHistory`](/match-score-history) engine driving the score logic.
  </Property>
  <Property name="winner" type="Team | None">
    Returns the [`Team`](/team) that won the match based on `best_of_sets`, or `None` if incomplete.
  </Property>
  <Property name="is_completed" type="bool">
    Returns `True` if a winner has been decided.
  </Property>
  <Property name="advantage_method" type="str">
    <AddedIn version="2.0.1">
      The game-level scoring rules. Can be `"advantage"` (standard advantage loop) or `"gold_point"` (golden point at 40-40).
    </AddedIn>
  </Property>
  <Property name="set_format" type="str">
    <AddedIn version="2.0.1">
      The format of sets played in the match. Can be `"standard"` (sets to 6 games) or `"mini"` (sets to 4 games).
    </AddedIn>
  </Property>
  <Property name="deciding_set_format" type="str">
    <AddedIn version="2.0.1">
      The format of the final/deciding set. Can be `"regular"` (standard set), `"tiebreak"` (7-point match tie-break), or `"super_tiebreak"` (10-point match super tie-break).
    </AddedIn>
  </Property>
  <Property name="starting_server_team" type="TeamId">
    <AddedIn version="2.0.1">
      The team configured to start serving first in the match.
    </AddedIn>
  </Property>
  <Property name="set_starting_servers" type="dict[int, dict[TeamId, Player]]">
    <AddedIn version="2.0.1">
      A dictionary mapping 0-based set indexes to each team's chosen initial server for that set.
    </AddedIn>
  </Property>
</Properties>

## Initialization

<Row>
  <Col>

    You can create a `Match` by providing optional parameters. If no `teams` are provided, default generic teams will be instantiated.

    ### Keyword Arguments

    <Properties>
      <Property name="teams" type="dict[TeamId, Team]" optional>
        A dictionary mapping a `TeamId` to a [`Team`](/team) object.
      </Property>
      <Property name="best_of_sets" type="int" optional default="3">
        The number of sets to determine a winner (e.g., `3` means the first to win 2 sets wins).
      </Property>
      <Property name="advantage_method" type="str" optional default="&quot;advantage&quot;">
        <AddedIn version="2.0.1">
          The game-level scoring system. Can be `"advantage"` (normal deuce/advantage) or `"gold_point"` (golden point at 40-40).
        </AddedIn>
      </Property>
      <Property name="set_format" type="str" optional default="&quot;standard&quot;">
        <AddedIn version="2.0.1">
          The length of sets. Can be `"standard"` (sets to 6 games) or `"mini"` (sets to 4 games).
        </AddedIn>
      </Property>
      <Property name="deciding_set_format" type="str" optional default="&quot;regular&quot;">
        <AddedIn version="2.0.1">
          How the final set is resolved if sets are tied. Can be `"regular"` (standard set), `"tiebreak"` (7-point tie-break), or `"super_tiebreak"` (10-point super tie-break).
        </AddedIn>
      </Property>
      <Property name="starting_server_team" type="TeamId" optional default="TeamId.A">
        <AddedIn version="2.0.1">
          The team that starts serving first in the match.
        </AddedIn>
      </Property>
      <Property name="set_starting_servers" type="dict[int, dict[TeamId, Player]] | None" optional default="None">
        <AddedIn version="2.0.1">
          A dictionary mapping a 0-based set index to a nested dictionary mapping `TeamId` to the `Player` chosen to start serving for that team in that set.
        </AddedIn>
      </Property>
      <Property name="duration_minutes" type="int | None" optional>
        Total duration of the match in minutes.
      </Property>
      <Property name="date" type="datetime | None" optional>
        The date and time of the match.
      </Property>
      <Property name="location" type="str | None" optional>
        The venue or location.
      </Property>
      <Property name="court_type" type="str | None" optional>
        E.g., "Indoor", "Outdoor".
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Instantiation" tag="class" label="Match">

    ```python
    from padelkit import Match
    from datetime import datetime

    # Start a premium mini-set match with gold points
    # and a match-deciding super tie-break
    match = Match(
        best_of_sets=3,
        advantage_method="gold_point",
        set_format="mini",
        deciding_set_format="super_tiebreak",
        location="Club Padel",
        date=datetime.now()
    )
    ```

    </CodeGroup>

    ```json {{ title: 'Output structure' }}
    Match(
      teams=...,
      best_of_sets=3,
      advantage_method='gold_point',
      set_format='mini',
      deciding_set_format='super_tiebreak',
      duration_minutes=None,
      date=datetime.datetime(...),
      location='Club Padel',
      court_type=None
    )
    ```

## Setting Set Starting Servers {{ tag: 'method', label: 'match.set_set_starting_server()' }}

<Row>
  <Col>

    Configures the starting server player for a specific team in a specific set.

    ### Required arguments

    <Properties>
      <Property name="set_index" type="int">
        The 0-based index of the set.
      </Property>
      <Property name="team_id" type="TeamId">
        The team identifying (`TeamId.A` or `TeamId.B`).
      </Property>
      <Property name="player" type="Player">
        The [`Player`](/player) chosen to serve first for the team in that set.
      </Property>
    </Properties>

  </Col>
  <Col sticky>

    <CodeGroup title="Usage" tag="method" label="set_set_starting_server">

    ```python
    from padelkit import Player, TeamId

    player = Player("Juan Lebrón")
    match.set_set_starting_server(set_index=0, team_id=TeamId.A, player=player)
    ```

    </CodeGroup>

  </Col>
</Row>
